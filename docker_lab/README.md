# Лабораторная работа 1 — Docker

Готовый вариант лабораторной: приложение на Python + Flask, база данных PostgreSQL, запуск через Docker Compose.

## Структура проекта

```text
.
├── Dockerfile
├── app
│   ├── app.py
│   └── requirements.txt
└── compose
    └── docker-compose.yaml
```

## Что делает приложение

Приложение запускается на порту `8080`.

Эндпоинты:

- `GET /` — проверка, что приложение работает;
- `GET /api/v1/get` — записывает запрос в PostgreSQL и возвращает список всех запросов.

## 1. Собрать образ

В корне проекта выполнить:

```bash
docker build -t DOCKERHUB_USERNAME/GROUP-INITIALS:latest .
```

Пример:

```bash
docker build -t ivanov/1-ii:latest .
```

## 2. Проверить, что образ собрался

```bash
docker images
```

## 3. Войти в Docker Hub

```bash
docker login
```

## 4. Загрузить образ в реестр

```bash
docker push DOCKERHUB_USERNAME/GROUP-INITIALS:latest
```

Пример:

```bash
docker push ivanov/1-ii:latest
```

После этого нужно сделать скриншот загруженного образа на Docker Hub и положить рядом с `Dockerfile`.

## 5. Исправить docker-compose.yaml

Откройте файл:

```text
compose/docker-compose.yaml
```

Замените строку:

```yaml
image: DOCKERHUB_USERNAME/GROUP-INITIALS:latest
```

на имя вашего образа, например:

```yaml
image: ivanov/1-ii:latest
```

## 6. Запустить приложение и базу данных

Из папки `compose` выполнить:

```bash
docker compose up -d
```

Проверить контейнеры:

```bash
docker ps
```

## 7. Проверить работу приложения

Открыть в браузере:

```text
http://localhost:8080/
```

Проверить запись запросов в базу:

```text
http://localhost:8080/api/v1/get
```

Каждый переход по `/api/v1/get` добавляет новую запись в таблицу PostgreSQL.

## 8. Остановить контейнеры

Из папки `compose`:

```bash
docker compose down
```

## Ответы на вопросы к защите

### Что такое контейнеризация?

Контейнеризация — это способ запускать приложение вместе со всеми зависимостями в изолированной среде. Благодаря этому приложение одинаково работает на разных компьютерах.

### Что такое Docker?

Docker — это платформа для создания, запуска и распространения контейнеров.

### Что такое образ?

Образ — это шаблон, из которого запускается контейнер. В образ входят приложение, зависимости и команда запуска.

### Как собрать образ?

Образ собирается командой:

```bash
docker build -t имя_образа .
```

### Что такое контейнер?

Контейнер — это запущенный экземпляр образа.

### Чем контейнер отличается от образа?

Образ — это как установочный шаблон, а контейнер — уже запущенное приложение на основе этого шаблона.

### Что такое реестр контейнеров?

Реестр — это хранилище образов. Например, Docker Hub. Туда можно загрузить образ, чтобы потом скачать его на другом компьютере.

### Как отправить образ в реестр?

```bash
docker login
docker push имя_образа
```

### Что такое docker-compose?

Docker Compose — инструмент для запуска нескольких контейнеров через один конфигурационный файл `docker-compose.yaml`.

### Как docker-compose управляет сетью?

Compose автоматически создаёт общую сеть для сервисов. Контейнеры могут обращаться друг к другу по имени сервиса. Например, приложение подключается к базе по адресу `postgres:5432`.

### Что такое проброс портов?

Проброс портов связывает порт внутри контейнера с портом на компьютере. Например:

```yaml
ports:
  - "8080:8080"
```

Это значит, что приложение внутри контейнера доступно на `localhost:8080`.
